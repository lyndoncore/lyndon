Image Optimizations

The image that I've included on this site, in its original form (from a stock photo site), was a 6500 x 5000 JPEG weighing in at 8.9MB (floral.jpeg). Although this is a super high-res image, and great for flexibility when designing, it's also a horrendously innapporpriate file to serve in production. This is not only because of the image's large file size, but also because this high of a resolution is simply not nessecary for its purposes on this site; ergo, some optimization is in order.

To make floral.jpeg a better fit for the web, and at the same time, accomplish its design purpose, it was converted to a PNG-8 in Photoshop CS6 (now floral.png). This reformatting not only reduces the file size, but preserves the desired detail within the image - as PNG makes use of lossless compression algorithms. Then, the file size of floral.png was further reduced by limiting the color palette to 128 colors, from 256 colors. This adjustment to the palette renders little perceivable difference in floral.png, but yields an additional 29.1% reduction in file size. By doing this, floral.jpeg has gone from 8.9MB to 4.0MB - an overall 55% savings.

Intuitively, the most direct image optimization is to just make the image smaller; so, floral.png was then pruned down to a reasonable resolution, from 6500 x 5000 to 1400 x 1077. Because floral.png is seamless, it could've afforded a much larger reduction in resolution, but the decision to keep it at 1400 x 1077 was based on the design objectives of the site, the contents of the image, and the image's engagement with accompanying text on the site. floral.png has now gone from a 8.9MB file to 377KB - an overall 95.8% savings

The last attempt at making the image as compact as possible, was to run it through OptiPNG - a command line "PNG optimizer that recompresses image files to a smaller size, without losing any information". Because floral.png had already been optimized through Photoshop, the additional savings from running OptiPNG's compression were only limited to 0.06%, or 239 bytes.
